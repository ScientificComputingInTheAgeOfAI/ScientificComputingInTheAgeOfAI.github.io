Scientific Computing in The Age of AI

Context

Artificial Intelligence (AI) is revolutionizing almost all aspects of our lives, and, in particular, the way in which we interact with computers. For instance, we are now at the point where an AI can write reasonably performant code in several programming languages, based only on a high-level natural language description of what the code should do [1, 2, 3]. As the quality and complexity of this AI-generated code keeps on growing, the role of the human programmer shifts from writing the actual code to instructing the AI and verifying its work. This paradigm shift promises to lower the cost of software and improve its quality. However, the effective use of these new technologies will require new skills, such as interaction with the AI, and verification of AI-generated code.

Goals

We aim to elucidate the consequences of code-generating AI for scientific programming and formulate a roadmap to optimally leverage the potential of these new technologies. The results of this roadmap (1) will be distilled into a high-impact publication and (2) will be used to develop training material for scientists and engineers at various levels.

Specifically, we want to bring together a panel of experts both from academia and industry to outline this roadmap for scientific programming in the age of AI. This roadmap should highlight both opportunities and challenges, and converge towards a set of best practices that will help guide scientists and engineers in future-proofing themselves amidst this paradigm shift. It is key that this panel includes the people who are developing these technologies, to ensure that the roadmap aligns with their visions and projected developments. The resulting training material will initially be provided in the UK by DiRAC, and in Belgium by the Flemish Supercomputer Centre (VSC), from which it can be distributed further to other organizations.

Suggested Topics

Below, we suggest some exemplar questions which could be discussed by the panel:

  • How can AI facilitate the general use of high-performance computing (HPC) resources?
  • How can AI be used to better document, test, and debug code?
  • Are the classical test-driven or behavior-driven development practices still suitable for AI-generated code? If not, how should they be adapted?
  • How can AI facilitate the use of heterogeneous computing resources? For instance, can an AI translate CPU code to performant GPU code?
  • How can AI help maintain essential legacy code?
  • How can AI be used for code optimization?
  • With the decreasing cost of software development due to AI, which otherwise unfeasible projects now come within reach?
  • Although the cost of software decreases, there are significant costs (e.g., environmental) and other issues (e.g., regarding intellectual property) associated with the development of the AI models driving these new technologies. How do we address or at least create awareness for these issues?
